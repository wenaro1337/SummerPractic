# Продуктовый магазин - база данных на Django

Учебное веб-приложение для ведения базы данных продуктового магазина. Проект позволяет управлять товарами, контролировать сроки хранения, выполнять уценку и анализировать продажи.

## Возможности

- просмотр товаров в общей таблице;
- добавление, изменение и удаление записей;
- поиск по коду и названию;
- фильтрация всех, просроченных, требующих уценки и наиболее продаваемых товаров;
- расчет остатка, даты окончания срока хранения и дохода;
- автоматическая уценка подходящих товаров на 30 процентов;
- определение самого продаваемого товара;
- построение гистограмм, круговой диаграммы и графика дохода;
- определение тенденции дохода и линейный прогноз на три месяца;
- безопасная выгрузка и загрузка товаров в формате SQLite;
- краткая подсказка по поиску, фильтрам и работе с базой.

## Данные товара

Таблица `store_product` содержит:

- код продукта;
- название;
- вид упаковки;
- дату поступления;
- срок хранения в днях;
- объем закупки;
- объем продажи;
- цену;
- признак выполненной уценки.

В рабочей базе проекта находится 100 демонстрационных товаров.

## Безопасная работа с базой

Выгрузка создает отдельный файл SQLite только с таблицей `store_product`. Пользователи Django, права доступа, сессии и другие служебные таблицы в экспорт не включаются.

При загрузке программа:

1. проверяет наличие и структуру `store_product`;
2. проверяет значения товарных записей;
3. сохраняет резервную копию текущих товаров в папке `backups`;
4. заменяет только товары в рабочей базе.

Сам файл `db.sqlite3` не подменяется, поэтому данные админ-панели не затрагиваются. Несовместимый или поврежденный файл отклоняется до изменения товаров.

## Установка

Клонировать репозиторий:

```text
git clone https://github.com/wenaro1337/SummerPractic.git
cd SummerPractic
```

Создать и активировать виртуальное окружение в Windows:

```text
python -m venv venv
venv\Scripts\activate
```

Установить зависимости:

```text
pip install -r requirements.txt
```

Применить миграции:

```text
python manage.py migrate
```

## Запуск

```text
python manage.py runserver
```

После запуска открыть:

- приложение: http://127.0.0.1:8000/;
- админ-панель: http://127.0.0.1:8000/admin/.

Учетная запись администратора создается отдельно командой:

```text
python manage.py createsuperuser
```

## Восстановление после удаления базы

Если файл `db.sqlite3` был удален, следует остановить сервер, выполнить миграции и снова запустить проект:

```text
python manage.py migrate
python manage.py runserver
```

После этого товары можно восстановить через интерфейс из ранее выгруженного файла.

## Структура проекта

- `magazin/` - настройки Django и основные маршруты;
- `store/models.py` - модель товара и вычисляемые показатели;
- `store/forms.py` - форма и проверка введенных данных;
- `store/views.py` - операции с товарами, импорт, экспорт и аналитика;
- `store/analytics.py` - расчеты, диаграммы, тенденция и прогноз;
- `store/errors.py` - сообщения с кодами E1xx, E2xx и E3xx;
- `store/templates/` - страницы пользовательского интерфейса;
- `db.sqlite3` - рабочая база Django;
- `requirements.txt` - зависимости проекта.
